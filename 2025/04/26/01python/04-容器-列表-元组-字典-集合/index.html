04-[容器]列表-元组-字典-集合

Posted on 2025-04-26

兄弟们，今天我们来聊聊 Python 里的“百宝箱”—— 列表和字典！📦✨ 它们就像我们收纳各种宝贝的小盒子，让我们的数据井井有条，方便又实用！快来一起解锁这两个超能力吧！😎💖

列表 (List)：我的有序“收藏夹”！<0xF0><0x9F><0x97><0x8F>️

列表就像一个可以存放很多东西的“收藏夹”，里面的每个“宝贝”都有自己的编号（索引），而且它们放进去的顺序是不会变的哦！就像我们衣橱里的衣服，按类别、按颜色整齐排列！🌈

列表里的每个元素都有一个索引，就像我们给每个衣服贴上标签，方便查找！索引从 0 开始计数哦！

“切”出一部分“收藏” (切片)：

列表也支持切片操作，可以一次性取出多个连续的“宝贝”！就像我们从一排书里拿出几本感兴趣的！📚

给我的“收藏夹”添砖加瓦 (添加元素)：

  • append()： 在列表的末尾添加一个新“宝贝”！就像我们衣橱里又添了一件新衣服！👗

  • insert()： 在列表的指定位置插入一个新“宝贝”！就像我们在衣橱的某个空位塞进一件心仪的小裙子！

  • extend()： 把另一个列表里的所有“宝贝”都添加到当前列表的末尾！就像我们把另一个小收纳盒里的东西都放进自己的大“收藏夹”！

把“不喜欢的”宝贝移走 (删除元素)：

  • pop()： 移除列表指定索引的“宝贝”，并且会告诉你移除的是哪个！就像我们从衣橱里拿走一件不常穿的衣服！

  • remove()： 移除列表中第一个匹配到的“宝贝”！就像我们从一堆同款衣服里找到一件不喜欢的扔掉！

  • del： 可以根据索引或者切片来删除列表中的“宝贝”！更直接更粗暴！💪

  • clear()： 清空列表里的所有“宝贝”！就像我们把整个衣橱都清空了！🧹

给我的“收藏”排排队 (排序)：

  • sort()： 对列表里的“宝贝”进行排序！默认是升序哦！就像我们整理衣服，按颜色深浅排列！

看看我的“收藏”有多少 (查找元素)：

  • index()： 查找指定“宝贝”在列表中第一次出现的位置！就像我们找一件特定的衣服在衣橱的哪个位置！

  • count()： 统计指定“宝贝”在列表中出现了多少次！就像我们数一下衣橱里有多少件白衬衫！

  • in / not in： 判断某个“宝贝”是否在我的“收藏夹”里！就像我们找一件衣服在不在衣橱里！

遍历我的“收藏” (循环访问)：

  • for 循环： 就像我们一个一个地拿出衣橱里的衣服欣赏！

  • while 循环： 也可以遍历列表，但需要自己控制索引！

  • enumerate()： 在遍历列表的同时告诉你当前“宝贝”的索引！就像我们拿出衣服的同时知道它是第几件！

字典 (Dictionary)：我的“标签小本本”！🏷️📒

字典就像一个“标签小本本”，里面的每个“宝贝”都有一个独一无二的“标签” (键)，通过这个“标签”我们可以快速找到对应的“宝贝” (值)！就像我们的通讯录，通过名字（键）快速找到电话号码（值）！📞

好的兄弟们，刚才我们聊了列表和字典这两个“收纳神器”，今天再给大家介绍两位新朋友——元组（Tuple）和集合（Set）！它们也各有妙用，能让我们的 Python “百宝箱”更加丰富多彩哦！💖✨

元组 (Tuple)：我的“只读收藏夹”！🔒

元组就像一个“只读”的列表！一旦创建，里面的“宝贝”就不能修改啦！就像我们珍藏的照片，记录了那一刻的美好，不希望被轻易改动！🖼️

查找“宝贝”的位置和数量：

元组也有 index() 和 count() 方法哦！

敲黑板！重点来了！ 元组里面的元素是不可变的！这意味着你不能像列表那样随意添加、删除或修改元素哦！🔒 这也是元组的一个重要特性，用于存储那些不希望被修改的数据！

集合 (Set)：我的“去重小助手”！<0xF0><0x9F><0xA7><0xAE>

集合就像一个不允许有重复元素的“袋子”！里面的“宝贝”都是独一无二的！而且集合里的元素是无序的哦！就像我们抓一把糖果，不会按照特定的顺序排列，而且相同的糖果只会算一个！🍬
