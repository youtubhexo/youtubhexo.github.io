磁盘分区操作

fdisk：用于管理 MBR 分区表的工具。 MBR传统分区表
gdisk：用于管理 GPT 分区表的工具。 GPT现代分区表
parted 相比于 gdisk (专门用于 GPT 磁盘) 和 fdisk (主要用于 MBR 磁盘)，parted 更加通用，可以处理多种分区表类型。它也提供了一个交互式界面和命令行模式，方便用户进行分区操作。

RAID 磁盘阵列

硬件 RAID：由专用的 RAID 控制卡处理，性能和管理通常较好，但成本较高。硬件 RAID 会在硬件层面上处理数据的冗余和校验。

软件 RAID：由操作系统控制，无需额外硬件。通常性能稍差，但成本低，配置和管理较为灵活，适合大多数应用场景。

优先级:

RAID 级别 性能 冗余性 存储空间利用率 适用场景 优缺点
RAID 0 提升读写性能，数据分布到多个磁盘并行处理 无冗余，单个磁盘故障导致数据丢失 100% 临时数据存储、高性能应用 优点：性能高，存储空间利用率高；缺点：无冗余，硬盘故障会导致数据丢失
RAID 1 读取性能提升，但写入性能无显著变化 数据完全镜像，单个磁盘故障数据不丢失 50% 需要高冗余的应用，如数据库服务器 优点：高冗余性，数据安全；缺点：存储空间利用率低
RAID 5 性能良好，读取性能高，写入性能较低（因奇偶校验） 提供冗余，通过奇偶校验信息恢复数据，单个磁盘故障不会丢失数据 （N-1）/N 需要平衡性能、冗余和存储空间的应用，如文件服务器 优点：冗余性好，性能较高，存储空间利用较好；缺点：写入性能稍低，奇偶校验计算开销
RAID 10 性能提升，结合了 RAID 1 的冗余和 RAID 0 的性能 提供冗余，至少两个磁盘故障（不同组）不会丢失数据 50% 高性能、高冗余要求的场景，如数据库和高可用服务器 优点：高性能，高冗余性，故障容忍度高；缺点：存储空间利用率低，至少需要 4 个磁盘

DNS的工作原理

自建DNS服务器

公司希望建立一个内部DNS服务器，通过设置对应的域名，采用内部DNS服务器负责转发，同时该服务器也要支持访问外部网络

DNS服务器调研

1、BIND (Berkeley Internet Name Domain): 这是最悠久、最广泛使用的 DNS 服务器软件。它功能强大且灵活，适用于各种规模的网络。BIND 支持各种 DNS 功能，包括权威 DNS 服务、递归 DNS 服务和缓存。
2、Unbound: 这是一款由 NLnet Labs 开发的开源 DNS 服务器。它以其速度和安全性而闻名。Unbound 是一款仅缓存的 DNS 解析器，这意味着它不提供权威 DNS 服务。
3、dnsmasq: 这是一个轻量级的 DNS 和 DHCP 服务器，易于配置。它适用于小型网络和嵌入式系统。dnsmasq 可以用作 DNS 转发器、DHCP 服务器和 TFTP 服务器。
4、PowerDNS: 这是一款高性能的 DNS 服务器软件，包括权威服务器和递归器。它适用于大型部署和需要高级 DNS 功能的环境。
5、Knot DNS: 这是一款高性能的权威 DNS 服务器，支持现代 DNS 功能，如增量区域传输 (IXFR) 和动态更新 (DDNS)。
6、CoreDNS: 这是一款用 Go 语言编写的灵活的 DNS 服务器，可以用作 Kubernetes 的 DNS 服务器。它具有插件架构，可以根据需要进行扩展。

1️⃣ FTP的AB面：主动模式 vs 被动模式

FTP有两种工作模式，就像我们逛街一样，有主动出击的“老店”，也有比较被动的“新店”。🤔

  • 主动模式： 端口固定是 21 (控制连接) 和 20 (数据连接)。服务器主动去连接客户端的数据端口。
  • 被动模式： 这就是“新店”啦！控制连接还是 21，但是数据连接的端口是由服务器临时开启并告诉客户端的。这种模式对于我们这些网络环境比较复杂的兄弟很友好，不容易被防火墙拦截哦！👍

  • get 是下载文件，put 是上传文件，mget 和 mput 前面加个 m，意思就是可以一次性下载/上传多个文件哦！

# 网络编程三要素：IP地址、端口和协议

在网络世界里，要找到一台特定的计算机并与之通信，就像我们寄快递需要地址一样，需要以下三个要素：

  • IP地址 (IP Address)：就像每台电脑在网络上的唯一地址，用于标识网络上的设备。
  • 端口 (Port)：一台计算机上可以运行多个网络程序，端口号就像是这些程序的不同房间号，用于区分不同的网络服务。
  • 协议 (Protocol)：网络通信需要双方遵守共同的“语言”和规则，这就是协议。例如，我们平时上网常用的 HTTP 就是一种协议。

今天我们主要关注 TCP协议，它是一种面向连接、可靠的协议，就像打电话一样，需要先建立连接才能进行数据传输，并且能保证数据传输的可靠性。

# 字符串的编码与解码

在网络传输中，数据通常以字节流的形式进行。而我们平时处理的文本是字符串。所以，在进行网络传输前，我们需要将字符串编码成字节串，接收到数据后，再将字节串解码成字符串。

常用的编码格式是 UTF-8，它可以支持各种语言的字符。

# Socket编程初体验

在Python中，我们可以使用 socket 模块进行网络编程。socket 可以看作是网络通信的一个端点，通过它可以发送和接收数据。

简单来说，TCP网络编程通常涉及以下步骤：

  1. 创建Socket：就像创建一根网线插头。
  2. 绑定IP地址和端口 (服务器端)：告诉别人你的“地址”和“房间号”，这样别人才能找到你。
  3. 监听连接 (服务器端)：等待别人的“电话”打进来。
  4. 接受连接 (服务器端)：有人打来“电话”了，接听一下。
  5. 发送/接收数据：开始“聊天”。
  6. 关闭连接：结束“通话”，挂断“电话”。

# 普通版本 vs Pro版本？

在网络编程中，可能会有不同版本的实现方式，例如基础版本可能只实现最基本的功能，而 Pro 版本可能会增加更多高级特性，比如并发处理、更复杂的协议支持等等。这就像我们使用的App，可能会有免费版和付费版，付费版功能更多更强大。

# 小贴士

  • 在进行网络编程时，要理解 IP 地址和端口的概念。
  • 数据在网络传输前需要进行编码，接收后需要进行解码，确保编码格式一致。
  • socket 模块是Python进行网络编程的核心工具。

# 告别崩溃！Python异常处理小技巧！🛡️

有没有遇到过程序突然报错停止的情况？那感觉简直太崩溃了！🤯 但是有了Python的异常处理机制，我们就可以优雅地捕获和处理这些错误，让程序即使遇到问题也能继续运行下去！是不是很神奇？✨

# 什么是异常？🤔

简单来说，异常就是在程序运行过程中发生的错误或意外情况。比如，除数为零啦，访问了不存在的列表索引啦，文件找不到啦等等。这些都会导致程序中断。

# try...except：捕获你的小错误！🎣

Python中使用 try...except 语句来捕获和处理异常。try 块中放置可能出错的代码，如果 try 块中的代码发生了异常，程序会跳转到对应的 except 块中执行处理代码。

  • 捕获单个异常

    上面的代码尝试进行除以零的操作，这会引发 ZeroDivisionError 异常。但是因为我们用了 try...except 捕获了这个异常，所以程序不会崩溃，而是会执行 except 块中的代码。

  • 捕获多个异常
    有时候一段代码可能会引发多种类型的异常，我们可以在一个 except 后面跟着多个异常类型，用逗号 , 分隔，或者使用多个 except 块来分别处理不同的异常。

    使用 as e 可以获取到具体的异常信息哦！

# else 和 finally：让你的异常处理更完整！🌟

除了 try 和 except，我们还可以使用 else 和 finally 块来完善异常处理。

  • else 块：当 try 块中的代码没有发生任何异常时，else 块中的代码会被执行。
  • finally 块：无论 try 块中的代码是否发生异常，finally 块中的代码都会被执行。通常用于进行清理操作，比如关闭文件、释放资源等。

# 异常的传递性 📢

如果一个函数内部发生了异常，并且没有被该函数处理，那么这个异常会向上层函数传递，直到被处理或者到达程序的顶层（导致程序崩溃）。

# 模块和包的导入

在Python中，我们可以通过 import 语句导入其他模块和包来使用别人写好的代码。这就像我们使用别人做好的工具一样，可以大大提高开发效率！

  • 导入模块：import 模块名
  • 导入包：import 包名.模块名

# 小贴士

  • 不要滥用异常处理，只捕获你期望处理的异常。
  • 尽量提供清晰的异常信息，方便调试。
  • finally 块非常重要，用于确保资源的正确释放。
