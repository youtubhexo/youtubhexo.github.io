06-传参

Python函数参数类型全解析，新手也能轻松掌握！🚀

哈喽各位小可爱！今天我们来聊聊Python函数中各种各样的参数类型，这可是写出灵活又强大的代码的关键哦！别担心，我会用最简单的方式带你搞懂！😎

# 认识你的函数朋友：不同类型的参数

在Python中，定义函数时，我们可以设置不同类型的参数，让函数能够接收各种各样的输入。主要有以下几种：

  • 位置传参（Positional Arguments）：这是最常见的参数类型啦！函数定义时的参数顺序很重要，你在调用函数时传入的值也会按照这个顺序一一对应。就像给你的朋友按名单顺序发礼物一样！🎁

  • 关键字传参（Keyword Arguments）：有时候我们记不清参数的顺序，或者想让代码更易读，就可以使用关键字传参。在调用函数时，直接指定参数名和对应的值。这样就不怕传错顺序啦！🔑

  • 默认值参数（Default Arguments）：在定义函数时，我们可以给一些参数设置默认值。如果在调用函数时没有给这些参数传值，它们就会使用默认值。就像你的朋友自带了某种小零食，如果你没带，就吃他的！🍬

  • 不定长位置参数（*args）：有时候我们不知道函数需要接收多少个位置参数，这时就可以使用*args。它会将传入的所有多余的位置参数打包成一个元组（tuple）。就像一个可以装很多东西的袋子！🎒

  • 不定长关键字参数（**kwargs）：类似地，**kwargs可以接收任意数量的关键字参数，并将它们打包成一个字典（dictionary）。每个传入的参数都需要指定参数名和值。就像一个可以存放各种标签和对应值的盒子！📦

# 划重点！参数使用的注意事项

  • 定义函数时，位置参数必须在默认值参数的前面。
  • 调用函数时，如果同时使用位置参数和关键字参数，位置参数必须在关键字参数的前面。
  • *args 和 **kwargs 通常放在参数列表的最后。

# 小彩蛋：匿名函数 lambda

除了用 def 定义函数，Python还提供了一种简洁的方式来创建匿名函数，也就是 lambda 函数。它通常用于一些简单的、只需要使用一次的函数。就像一个临时的便签！📝

兄弟们，今天我们来聊聊 Python 里的“循环”小宇宙！🌌✨ 它能让我们的代码像上了发条的小马达一样，重复执行我们想要的操作，简直是解放双手、提升效率的神器！😎💖

循环中的“俄罗斯套娃”！嵌套循环！🎁🎁🎁

想象一下，循环里面又套着一个循环，就像我们小时候玩的俄罗斯套娃一样！一层又一层，可以实现一些更复杂的逻辑哦！

遍历小能手！for 循环！🚶‍♀️🚶🚶‍♂️

for 循环就像一个贴心的导游，可以带着我们一个一个地“参观”容器里的每个“景点”（元素）。容器可以是列表、字符串等等，只要里面有东西，for 循环就能把它遍历个遍！

for 循环还能指定循环次数哦！用 range() 函数就可以实现啦！就像我们跑步，可以指定跑几圈！🏃‍♀️

永不停歇的小马达！无限循环！<0xF0><0x9F><0xAA><0xB0>

有时候我们希望程序一直运行下去，直到我们手动停止它，这时候就可以用 while True: 制造一个无限循环！就像一个永动机！

关键时刻喊“停”！break！✋

break 就像我们生活中的“紧急停止”按钮！在循环进行的过程中，如果遇到了 break，程序会立刻跳出整个循环，后面的代码也不会再执行啦！

“跳过”这个！continue！➡️

continue 就有点像我们生活中的“跳过”！在循环进行的过程中，如果遇到了 continue，程序会跳过当前这一次循环剩下的代码，直接进入下一次循环！

字符串的“门牌号”！索引！<0xF0><0x9F><0x97><0x8E>️

“切”出一块美味！切片！🔪🍰

切片就像我们切蛋糕一样，可以从字符串中“切”出一小块！用 : 来指定切片的范围！

字符串的“魔法棒”！常用操作！✨🪄

字符串还有很多自带的“魔法棒”，可以帮我们查找、修改、分割字符串！

  • 查找：find() 和 index()

  • 替换：replace()

  • 分割：split()

02-流程控制

Posted on 2025-04-26

兄弟们，今天我们来聊聊 Python 里的“流程控制”！🚦🚥 就像我们生活中的红绿灯，它能让我们的代码有条不紊地执行，该走哪条路，听谁的指令，都安排得明明白白！😎✨

让你的输出更精致！格式化输出小技巧！💅

有时候我们想让 print() 出来的文字更漂亮、更整齐，就需要用到格式化输出啦！就像我们化妆一样，精致一点才更吸睛！😉

  • 百分号 % 格式化： 这是 Python 比较“经典”的格式化方式啦！用不同的符号代表不同的数据类型：

    • %d：整数 (digit)
    • %f：浮点数 (float)
    • %s：字符串 (string)

  • F-string 格式化： 这是 Python 3.6 之后推出的超好用、超简洁的格式化方式！强烈推荐！👍 只需要在字符串前面加个 f，然后用大括号 {} 把变量包起来就好啦！简直不要太方便！💖

和你的程序“对话”！input() 输入函数！🗣️

我们的程序不能总是“自说自话”，有时候也需要我们用户输入一些信息，这时候就要用到 input() 函数啦！它就像一个“提问小助手”，等待我们输入内容。📝

敲黑板！重点来了！ input() 函数接收到的任何值都会变成字符串类型！即使你输入的是数字，它也会被当成文本来处理哦！所以，如果你想把输入的内容当成数字来用，就需要进行类型转换！

“七十二变”！数据类型转换！✨

有时候我们需要把数据从一种类型变成另一种类型，就像我们穿不同的衣服适应不同的场合一样！👗 Python 提供了一些内置函数来帮我们实现这个“变身”！

  • int(x)：把 x 转换成整数。注意！如果 x 是不能转换成整数的字符串（比如 “abc”），就会报错哦！⚠️
  • float(x)：把 x 转换成浮点数。
  • str(x)：把 x 转换成字符串。这个最常用啦，几乎所有类型都可以转换成字符串！

程序世界的“交通规则”！运算符！🚦

运算符就像我们生活中的各种符号，告诉程序该怎么“计算”或者“比较”不同的值。

  • 关系运算符： 用来比较两个值之间的关系，结果是布尔值（True 或 False）。就像我们比较谁高谁矮、谁大谁小一样！📏

    • > (大于)
    • < (小于)
    • >= (大于等于)
    • <= (小于等于)
    • == (等于)
    • != (不等于)

  • 逻辑运算符： 用来组合多个条件判断。就像我们说话时的“并且”、“或者”、“不是”！🔗

    • or (或者)：只要有一个条件为真，结果就为真。
    • and (并且)：所有条件都为真，结果才为真。
    • not (不是)：对一个条件取反，真变成假，假变成真。

  • 算术运算符： 用来进行数学计算。就像我们平时做的加减乘除！➕➖✖️➗

    • + (加)
    • - (减)
    • * (乘)
    • / (除)
    • // (整除，结果只保留整数部分)
    • % (取余数，就是除法运算后的余数)
    • ** (幂运算，就是几的几次方)

  • 赋值运算符： 用来给变量赋值。最常见的就是 = 等号啦！当然还有一些更简洁的写法：

    • = (等于)
    • += (加等于，相当于 a = a + b)
    • -= (减等于，相当于 a = a - b)
    • *= (乘等于，相当于 a = a * b)
    • /= (除等于，相当于 a = a / b)
    • //= (整除等于)
    • %= (取余等于)
    • **= (幂等于)

让你的代码“聪明”起来！if 语句！🧠

if 语句就像我们生活中的“如果…就…”的判断。它能让我们的代码根据不同的条件执行不同的操作，变得更智能！

  • if 单分支结构： 如果条件成立，就执行一些代码。

  • if 双分支结构： 如果条件成立，执行一些代码；否则，执行另一些代码。就像我们选择题，要么 A，要么 B！

  • if 多分支结构： 有多个条件需要判断的时候用！就像我们根据不同的分数评判等级一样！

让你的代码“循环”起来！while 语句！🔄

while 循环就像我们生活中的“重复…直到…”的过程。只要条件成立，它就会一直执行一些代码，直到条件不成立才停止。

07-系统服务管理

Posted on 2025-04-26

兄弟们，今天来聊聊 Linux 系统里那些让电脑“乖乖听话”的小秘密！🤫✨ 掌握了这些，你也能轻松安排电脑的日常工作啦！💻💖

让服务跑起来！systemctl 大法！🚀

在 Linux 这个神奇的世界里，各种功能都像一个个小小的“服务”在后台默默运行。比如你想远程连接你的电脑？那就需要 sshd 这个“小助手”一直在后台待命！怎么让它们启动、停止、重启，甚至查看状态呢？就靠我们强大的 systemctl 命令啦！💪

就像遥控器一样，systemctl 可以控制这些“服务”的状态：

  • 启动服务： sudo systemctl start <服务名> 比如启动我们的远程连接小助手：sudo systemctl start sshd 启动后你的电脑就能接受远程连接啦！🔗
  • 停止服务： sudo systemctl stop <服务名> 暂时不需要某个服务了？就让它先休息一下：sudo systemctl stop sshd
  • 重启服务： sudo systemctl restart <服务名> 有时候服务出点小状况，重启一下就好啦！就像给 App “刷新”一下：sudo systemctl restart sshd
  • 查看服务状态： sudo systemctl status <服务名> 想知道某个服务现在正不正常工作？看看它的状态就知道啦！会显示运行状态、进程信息等等，超详细！🔍

定时任务？交给 crontab 准没错！⏰

有时候我们希望电脑在特定的时间自动帮我们做一些事情，比如每天凌晨备份数据、定时清理临时文件等等。这时候就要请出我们的“定时任务小管家”—— crontab 啦！🕰️

你可以编辑一个叫做 “crontab” 的文件，告诉电脑在什么时间执行什么命令。

编辑你的定时任务列表：

第一次运行可能会让你选择一个编辑器，一般选 nano 就很友好啦！😊

crontab 的时间格式是这样的：

每个字段的取值范围都贴心地告诉你啦：

  • 分： 0-59
  • 时： 0-23
  • 日： 1-31
  • 月： 1-12
  • 周： 0-7 (0 和 7 都表示星期天哦！)

一些超实用的小技巧：

  • *： 表示这个字段的每一个值都符合。比如 * * * * * 就表示每分钟都执行后面的命令！😱
  • -： 表示一个连续的范围。比如 1-7 就表示 1 到 7。
  • ,： 表示多个不连续的值。比如 1,2,6 就表示 1、2 和 6。
  • /： 表示每隔多久执行一次。比如 0/10 在“分”的字段表示每隔 10 分钟执行一次（也就是 0, 10, 20, 30, 40, 50 分）。

举几个超简单的定时任务栗子🌰：

  • 每天凌晨 3 点备份我的重要文件： 0 3 * * * /bin/bash /path/to/backup_script.sh
  • 每个工作日的下午 5 点发送一封邮件提醒自己下班： 0 17 * * 1-5 /usr/bin/mail -s "下班啦！" your_email@example.com < /dev/null
  • 每隔 30 分钟检查一下服务器状态： */30 * * * * /path/to/check_server_status.sh

编辑完 crontab 文件记得保存哦！电脑就会乖乖地按照你的时间表去工作啦！是不是感觉自己像个小小的“时间魔法师”？🧙‍♀️✨

06-软件安装和系统服务

Posted on 2025-04-26

兄弟们，今天来聊聊在 Linux 系统上怎么“捯饬”各种软件和系统服务！💻✨ 别看命令行黑乎乎的，玩熟了简直打开新世界的大门！😎

软件分类：你的 App “出生”方式大揭秘！👶➡️🚀

话说软件就像我们的小可爱 App，它们的“出生”方式可不一样哦！主要分两种：

1. 源码包： 就像拿到 App 的“源代码”！📜 你需要自己配置好“开发环境”（各种依赖），然后像个小码农一样“编译”它，再“打包”成能安装的“安装包”，最后才能安装使用。听起来是不是有点硬核？💪 但好处是你可以根据自己的需求定制功能，就像自己 DIY 一个独一无二的手机壳！🎨

举个栗子🌰： 很多超 cool、最新的开源软件都是以源码包的形式发布的。比如你想用某个还没被主流 Linux 发行版收录的黑科技工具，很可能就需要你先下载源码，然后 ./configure -> make -> make install 三连击！💥

2. 二进制包： 这就像直接拿到已经打包好的 App 安装包（比如安卓的 .apk，iOS 的 .ipa）。你不需要关心它背后的“源代码”是啥，直接“一键安装”就能用！✌️ 这种方式为了保护开发者的知识产权，会进行代码混淆，让别人不容易直接看到源码。

二进制包又分离线安装和在线安装两种方式，各有千秋，适用不同场景哦！

RPM 离线安装：网不好？没关系！📦

RPM（Red Hat Package Manager）是很多 Linux 发行版（比如 CentOS、Fedora）都在用的“快递”格式！📦 你可以提前下载好 .rpm 包，然后在没网或者网络不好的情况下直接安装！

常用命令小贴士：

  • 查询已安装的 RPM 包： rpm -qa （这个命令会列出所有已安装的 RPM 包，信息量有点大哈！👀）
  • 精准查找包含特定关键词的包： rpm -qa | grep <关键词> 比如你想找包含 wget 的包：rpm -qa | grep wget
  • 查看特定 RPM 包的详细信息： rpm -qi <包名> 比如：rpm -qi wget-1.21.1-8.el9.x86_64
  • 安装本地 RPM 包： rpm -ivh <.rpm 包文件> i 是安装，v 是显示详细信息，h 是显示安装进度小方块，敲起来是不是很有仪式感？🧱
    • 强制安装（小心使用！）： 有时候遇到依赖问题，可以尝试加上 --nodeps --force，但可能会导致软件运行不稳定哦！⚠️
  • 升级 RPM 包： rpm -Uvh <.rpm 包文件> U 是升级，如果没安装过就相当于安装。
  • 卸载 RPM 包： rpm -e <包名> 注意这里是包名，不是 .rpm 文件名！比如卸载 wget：rpm -e wget
    • 强制卸载（同样小心！）： 同样可以加上 --nodeps --force，但后果自负哦！💣

DNF 在线安装：冲浪达人的必备技能！🏄‍♀️

DNF（Dandified YUM）是新一代的包管理器，在 Fedora 和一些新的 CentOS 版本上取代了 YUM。它就像一个智能的“App Store”，连接着软件仓库，可以自动帮你下载、安装、升级、卸载软件，还能自动处理依赖关系，简直是懒人必备！😎

常用命令速记：

  • 列出所有可用的软件包： dnf list （信息量巨大！🐳）
  • 搜索软件包： dnf search <关键词> 比如你想找关于 Python 的包：dnf search python
  • 查看软件包信息： dnf info <包名> 比如：dnf info python3
  • 安装软件包： dnf install <包名> 比如安装 Python：dnf install python3 它会自动帮你下载并安装所有相关的依赖包，超省心！💖
  • 升级软件包： dnf upgrade <包名> 升级指定的包，比如升级 Python：dnf upgrade python3
  • 升级所有软件包： dnf update 或者 dnf upgrade （推荐定期执行，保持系统软件最新！✨）
  • 移除软件包： dnf remove <包名> 比如移除 Python：dnf remove python3
    • 移除不再需要的依赖包： dnf autoremove 卸载掉那些因为之前安装某个包而自动安装，但现在那个包已经被卸载了的“孤儿”依赖包，让你的系统更干净！🧹
  • 列出已安装的软件包： dnf list --installed 想看看自己都装了啥？用这个！🧐
  • 查找已安装的包含特定关键词的包： dnf list --installed | grep <关键词> 比如查找已安装的 Python 包：dnf list --installed | grep python
  • 清理缓存： dnf clean all 有时候遇到问题可以尝试清理一下缓存。

举几个 DNF 的实用例子🌰：

  • 想安装一个超好用的文本编辑器 nano： sudo dnf install nano
  • 发现 vim 太难学，想换回 nano？ sudo dnf remove vim
  • 听说 neofetch 可以酷炫地展示系统信息，想试试？ sudo dnf install neofetch 然后运行 neofetch 看看效果！🤩
  • 系统提示有更新，想一次性升级所有软件？ sudo dnf update -y （加上 -y 可以自动确认，不用每次都按 y 啦！😉）

好啦，关于 Linux 下的软件安装就先聊到这里！是不是感觉也没那么难嘛？😉 掌握了这些小技巧，你也能轻松管理你的 Linux 系统啦！下次再和兄弟们分享更多好玩的技术干货！💖✨

05-权限管理

Posted on 2025-04-26

🌟 Linux 权限管理新手小白也能看懂！告别文件“锁🔒”烦恼！

兄弟们，今天给大家分享一下在 Linux 系统里超级重要的一个知识点——权限管理！💻💾 别看它好像很高深的样子，其实掌握了它，就能更好地保护我们的文件和数据哦！💪

🤔 权限是个啥？

在 Linux 这个神奇的世界里，每个文件和文件夹都有自己的“主人”和“管家”，还有一些“访客”。权限就像是给他们设置的“通行证”，规定了谁能看👀、谁能动✍️、谁能用▶️！

简单来说，权限就三种：

  • r (Read - 读)：👀 可以看里面的内容！
    • 文件：就能打开看到文件写了啥。
    • 目录：就能看到这个文件夹里都有哪些文件和文件夹的名字。
  • w (Write - 写)：✍️ 可以修改里面的内容！
    • 文件：就能编辑、保存文件里的东西。
    • 目录：就能在这个文件夹里新建、删除、移动文件和文件夹！⚠️ 小心操作哦！
  • x (Execute - 执行)：▶️ 可以运行它！
    • 文件：对于程序或者脚本文件，有了这个权限才能跑起来！
    • 目录：有了这个权限才能用 cd 命令进入这个文件夹！🚪

🔍 权限信息在哪里看？

在终端里输入 ls -l 命令，就能看到文件和文件夹的详细信息啦！就像这样：

最前面那一串像 drwxr-xr-x 和 -rw-r--r-- 的就是权限信息啦！🤩

  • 第一位：是文件类型哦！- 代表普通文件，d 代表文件夹，l 代表快捷方式。

  • 后面九位：分成三组，每组三个字母，分别代表 文件主人 (User)、文件所属组 (Group)、其他人 (Others) 的权限！

✍️ 怎么修改权限？两种方法安排上！

🅰️ 字母大法：简单易懂！

用 chmod 命令就可以修改权限啦！

  • u：主人
  • g：所属组
  • o：其他人
  • +：添加权限
  • -：移除权限
  • =：设置成某个权限

比如，我想给我写的 linux.txt 文件加上执行权限，让它跑起来！🏃‍♀️

如果想给一个文件夹和它里面的所有东西都加上某个权限，记得加上 -R 参数哦！🔄

🔢 数字密码：更快更直接！

还有一种更酷的方法，用数字来代表权限！😎

  • r = 4
  • w = 2
  • x = 1

把想要的权限数字加起来就好啦！

  • 7 (4+2+1) = rwx (读写执行全都有！)
  • 6 (4+2) = rw- (可以读写，不能执行)
  • 5 (4+1) = r-x (可以读和执行，不能写)
  • 4 = r-- (只能读)

比如，我想给 shop 这个文件夹所有权限！安排！👇

⚠️ 777 权限要谨慎使用哦，因为它给所有人开了大门！🚪

🤫 默认权限的小秘密 (umask)

每次我们新建文件或者文件夹的时候，系统都会给它们一个默认的权限。这个默认权限是由 umask 这个小家伙决定的！🤫 虽然笔记里没细说，但记住有这么个东西就好啦！

✨ 特殊权限了解一下！

Linux 还有一些特殊的“小彩蛋”权限，虽然不常用，但是了解一下总没错！

  • 冒险位 (SUID & SGID)：🚀 让普通用户也能临时拥有文件主人或者文件所属组的“超能力”！
    • S：表示设置了冒险位，但主人/组没有执行权限（不常见）。
    • s：表示设置了冒险位，并且主人/组有执行权限（更常见）。
  • 粘滞位 (Sticky Bit)：📍 给公共文件夹加个“小门卫”，只有文件的主人、文件夹的主人或者 root 大佬才能删除里面的文件，别人想动？没门！🙅‍♀️ 用 T 表示哦！

🤝 文件和文件夹的“户口本”

  • 所属用户 (chown)：可以修改文件和文件夹的“主人”！🏠
  • 所属组 (chgrp)：可以修改文件和文件夹的“家庭”！🏘️

🛡️ 更高级的权限管理 (ACL)

如果觉得基本权限不够用，还有更厉害的 ACL (Access Control List)！它可以针对** конкретный 用户** 设置更细致的权限！📝 就像给 VIP 用户开通专属通道！下次有机会再给大家细讲！

🧑‍🤝‍🧑 用户和用户组小课堂

  • groupadd 用户组名称：新建一个“家庭”！
  • groupmod 用户组名称：修改“家庭”信息。
  • groupdel 用户组名称：解散“家庭”。
  • /etc/group：这里记录了所有“家庭”的信息哦！
  • useradd 用户名称：新添一个“家庭成员”！
  • usermod 用户名称：修改“家庭成员”信息。
  • userdel 用户名称：移除“家庭成员”。
