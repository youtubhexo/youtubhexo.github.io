02-Linux文件及目录操作：我的 Linux 基础笔记 📂

Posted on 2025-04-25

Linux 目录结构：系统的骨架 🦴

在 Linux 系统中，目录的组织结构是理解文件操作的基础。以下是一些重要的目录及其作用：

  • /usr：系统软件默认安装位置，通常包含用户安装的应用程序和库文件。（思考：为什么不直接安装在根目录下？）
  • /opt：可选软件包目录，通常用于安装第三方或大型商业软件。（思考：将第三方软件放在这里有什么好处？）
  • /etc：系统配置文件目录，几乎所有系统服务和应用程序的配置都在这里。（重要！修改配置文件需要谨慎。）
  • /dev：设备文件目录，Linux 将所有硬件设备都抽象为文件。（思考：如何通过操作这些文件与硬件交互？）
  • /tmp：临时文件存储目录，系统重启后该目录下的文件通常会被清空。（注意：不要在这里存放重要数据。）
  • /home：普通用户的家目录，每个用户都有自己的独立空间。（我的个人文件通常在这里。）
  • /root：超级管理员（root）的家目录，拥有系统的最高权限。（谨慎使用 root 权限！）

Linux 目录相关命令：我的导航工具 🧭

学习这些命令，就像学会了在 Linux 世界中导航：

  • pwd：显示当前工作目录（Print Working Directory）。（我当前在哪儿？）
  • ls：列出目录内容（List）。
    • ls -a：显示所有目录和文件，包括以 . 开头的隐藏文件。（隐藏文件有什么用？）
    • ls -l：以长列表格式显示，包含详细的文件权限、所有者、大小和修改时间等信息。（这些信息的含义是什么？）
    • ls -h：以人类可读的格式显示文件大小（Human-readable）。（比如 KB、MB、GB，更直观。）
  • mkdir <目录名>：创建新目录（Make Directory）。
  • rmdir <空目录名>：删除空目录（Remove Directory）。（只能删除空目录，不常用。）
  • cd <目标目录>：切换工作目录（Change Directory）。
    • cd ..：返回上一级目录。
    • cd ~ 或 cd：返回当前用户的家目录。
  • tree <目录名>：以树状结构显示目录内容。（清晰地展示目录层级关系，很直观。）

Linux 文件相关命令：我的操作利器 🔪

这些命令用于对文件进行各种操作：

  • 创建文件：

    • touch <文件名>：创建一个空文件或更新文件的时间戳。（很常用的创建空文件的方法。）

  • 复制文件：

    • cp -r <源文件/目录> <目标路径>：复制文件或目录（Copy，-r 用于复制目录，表示递归复制）。（复制目录时一定要加 -r！）

  • 移动文件：

    • mv <源文件/目录> <目标路径>：移动文件或目录，也可以用于重命名（Move）。（如果目标路径是已存在的目录，则移动到该目录下；如果是新的文件名，则重命名。）

  • 文件上传与下载：（思考：有哪些常用的文件上传和下载工具？例如 scp、sftp、rz/sz 等。）

  • 删除文件：

    • rm -r <文件/目录>：删除文件或目录（Remove，-r 用于删除目录，请谨慎使用！）。（删除操作不可逆，一定要小心！）

  • 文本查找命令：在浩瀚的文件中寻找目标 🔍

    • find <查找路径> <查找条件>：强大的文件查找工具。

      • -name <文件名>：按文件名进行搜索（支持通配符 *）。
      • -type f：查找文件类型为普通文件（d 表示目录，l 表示链接等）。
      • -size <+/-><大小>：按文件大小搜索，+ 大于，- 小于。（例如 +100M 表示大于 100MB。）
      • -mtime <+/-><天数>：按文件修改时间搜索，+n 表示 n 天以前修改过，-n 表示 n 天以内修改过。
      • -exec <命令> {} \;：对查找到的文件执行指定的命令，{} 代表查找到的每个文件，\; 是命令结束符。（非常灵活的操作！）

    • 举例： find /usr/lib -size +100M -type f -exec cp {} /archive/large_files \;

      这条命令的含义是：在 /usr/lib 目录下查找所有大于 100MB 的普通文件 (-type f -size +100M)，并将找到的每个文件 ({}) 复制 (cp) 到 /archive/large_files 目录下，直到遇到 \; 结束。

01-基础扫盲工作 - 我的运维学习笔记 ✍️

Posted on 2025-04-25

初探运维：我的理解 🤔

一开始，我对运维的印象可能比较模糊，但通过学习，我逐渐认识到：

  1. 核心定位：我们是软件运维，专注于软件系统的稳定运行。这和硬件运维有所区分。
  2. 关键价值：保障业务连续性。运维不仅仅是修机器，更重要的是通过专业技术，确保公司的 IT 系统像血液一样流畅运转，保障业务不中断。
  3. 意外的惊喜：薪资潜力超乎想象！有经验的运维高手，在薪资上竟然能超越开发、测试甚至 UI，这让我对这个领域刮目相看。

运维日常：我在做什么？ 🛠️

那么，运维具体都在做些什么呢？我的理解是：

  1. 像个细心的管家：系统的监控与维护。我们需要时刻关注服务器、网络设备的健康状况，就像给它们做体检，确保一切正常。（思考：有哪些常用的监控工具呢？）
  2. 化身救火队员：故障的排查与处理。当系统出现问题时，我们要快速找到原因并解决它，这需要冷静的头脑和扎实的技术。（思考：遇到不同类型的故障，排查思路有什么不同？）
  3. 追求效率的极客：自动化运维。重复性的工作应该交给机器去做，编写自动化脚本就是解放双手，提高效率的关键。（思考：哪些运维任务适合自动化？）
  4. 系统的搭建者和升级者：系统的部署与更新。根据业务需求，我们需要部署新的系统，并对现有系统进行升级。（思考：部署和升级过程中有哪些关键步骤和风险点？）
  5. 数据的守护者：备份与恢复。数据是企业的生命线，定期备份和制定恢复方案至关重要。（思考：有哪些常见的备份策略？）
  6. 安全的第一道防线：安全管理。及时发现和修复系统及应用的安全漏洞，防止黑客入侵。（思考：如何及时获取和应对安全漏洞信息？）

云计算：未来的趋势？ ☁️

云计算这个概念很火，我的理解是它带来了极大的便利性：

  • 随时调整操作系统，就像换手机系统一样灵活。
  • CPU 和内存资源、硬盘、网络带宽都可以根据需求动态调整，特别是阿里云的 100M 带宽，简直是飞一般的感觉！
